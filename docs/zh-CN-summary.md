## README 摘要

本仓库是一套自主 LLM agent 执行治理系统的披露包,系统于 2026 年 5-7 月间
私下构建并实际使用。包内包含设计思想(essays/)、机制规格(specs/)与可核
验的开发年表(CHRONOLOGY.md);实现代码有意不公开(分级见
DISCLOSURE-MAP.md)。

问题层次:prompt engineering 塑造问什么,context engineering 塑造答题时
知道什么,harness engineering 塑造 agent 借以行动的机械,loop engineering
塑造长任务如何跨轮持续。这些之上还有一层未被命名的问题:**自主运行凭什么
可信**——不是 agent 能不能干活,而是它对工作的陈述是否为真、计划接触现实
后发生了什么、以及管它的规则本身是否被管理。

本包记录的机制:两道阻塞式审计门(计划先审后执行、执行先审后收工,由运行
时钩子强制而非口头约定);言行对账(六项机器检查,对比状态"说了什么"与工
件"显示什么");四格偏差分类法(按计划/绕行/硬啃/上报——按偏差**许可的
响应**分类);证据分级状态(弱/中/强,收工需中级以上证据);自征税的规则
生命周期(每条规则必须付租,晋级需 owner 打勾,存疑机器进缓刑,死规则退休)。

状态:已备好发布,**尚未发布**。发布、定名、许可证确认、以及任何实现层的
公开均为 owner 独有决定。
